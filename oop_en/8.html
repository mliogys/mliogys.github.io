Interfeisas apibrėžia kokias operacijas klasė privalo vykdyti, bet ne taip kaip tos operacijos turi būti vykdomos. Interfeisas aprašomas labai panašiai kaip klasė, tik operacijos (metodai) pateikiamos be realizacijos. Panagrinėkime pavyzdžius.

Interfeiso sintaksė:
1.interface InterfaceName
2.{
3.// metodų antraštės
4....;
5.}
Interfeiso pavyzdys:
1.interface IVaccinable
2.{
3.void Vaccine();
4.}
Išvestinė klasė Dog:
1. class Dog : Animal, IVaccinable
2. {
3. public Dog(string breedTitle)
4. {
5. this.breedTitle = breedTitle;
6. }
7.
8. public override void MakeSound()
9. {
10.WriteLine("Woof Woof");
11.}
12.public override void Vaccine()
13.{
14.WriteLine("Nobivac vaccine");
15.}
16.}
Išvestinė klasė Cat:
1. class Cat : Animal, IVaccinable
2. {
3. public Cat(string breedTitle)
4. {
5. this.breedTitle = breedTitle;
6. }
7.
8. public override void MakeSound()
9. {
10.WriteLine("Meow Meow");
11.}
12.public override void Vaccine()
13.{
14.WriteLine("Rabies vaccine");
15.}
16.}
Objektai:
1. Dog dog = new Dog("German Shepherd");
2. Cat cat = new Cat("Persian Longhair");
3.
4. dog.MakeSound(); // Woof Woof
5. cat.MakeSound(); // Meow Meow
6. dog.Vaccine(); // Nobivac vaccine
7. cat.Vaccine(); // Rabies vaccine

Pastebėkime, jog apibrėždami metodų rinkinį nenurodėme matomumo lygio (public) bei raktinio žodžio abstract. Taip yra dėl to, jog apibrėžiant interfeisą ši ypatybė automatiškai yra priskiriama ir ji negali būti pakeista. Klasė interfeisų gali realizuoti neapibrėžtą kiekį.

Python kalboje interfeisų nėra, nes šioje kalboje yra galimas daugybinis paveldėjimas ir interfeisai, tokiu atveju, neturi prasmės.
