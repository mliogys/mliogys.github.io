Inkapsuliacija (angl. encapsulation) vienas pagrindinių objektinio programavimo principų, leidžiantis paslėpti vidinę objekto konstrukciją ir išlaikyti duomenų integralumą.

Panagrinėkime klasę, parašytą C# kalba.

1. class Building
2. {
3. public int numberOfRooms;
4. public int numberOfFloors;
5.
6. public Building()
7. {
8. numberOfRooms = 1;
9. numberOfFloors = 1;
10.}
11.}

Problema su tokiu klasės apibrėžimu yra ta, kad klasės atributai laisvai prieinami iš išorės ir gali būti laisvai modifikuojami, priskiriant nekorektiškas reikšmes. Pavyzdžio atveju, klasės atributams numberOfRooms ir numberOfFloors gali būti priskirta bet kokia reikšmė intervale [–2 147 483 648; 2 147 483 647], o tai ne visais atvejais yra logiška, nors duomenų tipas tai leidžia.

Modifikatorius public nėra tinkamas modifikatorius atributams, kuriems reikia apibrėžti tam tikras taisykles. Siekiant apsaugoti klasės narius iš išorės reikia rinktis private arba protected modifikatorius. Apžvelkime modifikatorių apibrėžimus:

  • public - klasės narys prieinamas iš klasės vidaus ir išorės.
  • private - klasės narys prieinamas tik iš klasės, kurioje jis apibrėžtas, vidaus.
  • protected - klasės narys prieinams tik iš klasės, kurioje jis apibrėžtas, vidaus ir išvestinės klasės vidaus. Plačiau apie tai, 5 skyriuje Paveldėjimas.

Panagrinėsime pavyzdžius, bet prieš tai, verta paminėti, jog Python kalboje nėra modifikatorių, visi klasės nariai vieši. Python kalbos principas - kalba, žinantiems programuotojams ką daro. Pavyzdyje pastebime, kad pakeitus atributų modifikatorius iš public į private, atsirado papildomi public metodai: Get ir Set. Tai yra būdinga inkapsuliacijos atveju. Get metodas grąžina atitinkamo atributo reikšmę, o Set metodas - modifikuoja. Paprastai, pastarajame metode yra aprašoma verslo logika, neleidžianti priskirti bet kokių, nesąmoningų reikšmių.
