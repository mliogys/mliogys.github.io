Exception handling is part of programming routine, used to control situations that can crash software during its runtime. Exception can appear for many reasons, such as wrong user input, incorrect arithmetic operations (for example, division by zero), accessing protected memory, writing to read-only file and so on. If exceptions are not considered and software end up meeting those, software will stop working and will crash.

For successful exception handling, these four things are needed to consider and implemented:

  • Object carrying exception details.
  • Operator throwing exception to caller object.
  • Block for catching exception.
  • Exception overtake.

First two things must be implemented in situations that can create exception, last two things are needed to manage appeared exception. Let's take a look at the example.

As we can see in example No. 2 exception handling in caller side is implemented using try-catch block (C# and Java) or try-except (Python). Code part that can end-up in error (exception) must be written in try block, and actions to handle error in catch (except) block. Code in try block is executed while no exception is met, then the execution is passed to catch (except) block.
