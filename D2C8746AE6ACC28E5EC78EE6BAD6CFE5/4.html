Encapsulation is one of the three major object-oriented programming principles, enabling us to hide internal object construction and to ensure, at the same time, data integrity.

Let's look at (C#) class definition.

1. class Building
2. {
3. public int numberOfRooms;
4. public int numberOfFloors;
5.
6. public Building()
7. {
8. numberOfRooms = 1;
9. numberOfFloors = 1;
10.}
11.}

At first glance, everything is right with class definition, but problem is such, that class attributes are accessible directly from the outside (another class) and can be easily modified by assigning business logic violating values. For example, attributes numberOfRooms and numberOfFloors can have any value from interval [â€“2 147 483 648; 2 147 483 647] assigned, but not all values make sence or meet bussiness rules..

Modifier public is not the right choice for attributes with restrictions in values. In order to protect class members form direct access we have to choose private or protected modifiers. Modifiers definition:

  • public - class members accessible from inside and outside the class.
  • private - class members accessible only from the inside of a class it is defined.
  • protected - class members accessible only from the inside of a class it is defined and all derived ones. More about this modifier will be in Chapter No. 5 Inheritance.

In last example we can notice that a few additional methods were added after declaring attributes private. These methods are call getters (returning attribute value) and setters (setting attribute value). Ususally, setter method is a method there all business logic is applied and data validation is performed.
