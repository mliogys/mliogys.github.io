Object-oriented Programming - programming paradigm, based on objects (special kind of data structures with predefined operations) processing. Object-oriented programming history starts from 1967 and during all the years there are 5 key principles crystalized:

  • Everything is an object. Object is a composite data structure storing data (attributes) and having data processing algorithms (methods) letting manage data from inside of an object itself.
  • Program - is a whole of interconnected objects. Objects communicate with each other by calling each other's methods. Method call can be also treated as sending message.
  • Each object has allocated memory space in RAM.
  • Each object is an instance of some kind (type). Type (class) defines what kind of messages object will be able to receive and process.
  • All objects of the same origin can accept same messages. This principle is one of key principles and it will be discussed in chapter No. 5 Inheritance and chapter No. 6 Polymorphism.
