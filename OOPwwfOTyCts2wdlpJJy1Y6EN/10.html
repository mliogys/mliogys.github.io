Objects collections can be categorized to: static and dynamic collections. Static collections are fixed in size, which is set during initialization and it can not be changed during the application runtime. Dynamic collections are not fixed in size and it can be changed during the application runtime. Typical representation of static collections is array type collection. Let's look at the example.

Dynamic collections are not fixed in size. If needed, they can be expanded by adding new objects, or they can be reduzed by removing objects. Let's look at the example.

List initialization:
1. // List filled with values during the initialization
2. List<int> integersList = new List<int>() { 5, 6, 7, 2, 6 };
3.
4. // List initialized first and later filled with actual values
5. List<int> integersList = new List<int>();
6. integersList.Add(5);
7. integersList.Add(6);
8. integersList.Add(7);
9. integersList.Add(2);
10.integersList.Add(6);
11.
12.// Objects list initialization
13.List<Circle> circlesList = new List<Circle>();
14.circlesList.Add(new Circle(8.5));
15.circlesList.Add(new Circle(2.4));
16.circlesList.Add(new Circle(5.1));
17.circlesList.Add(new Circle(6.4));
18.circlesList.Add(new Circle(3.7));
Sąrašo valdymas:
1. // List size
2. int n = integersList.Count;
3.
4. // Iteration through list using for
5. for (int i = 0; i < circlesList.Count; i++)
6. {
7. WriteLine($"Area Size = {circlesList[i].GetAreaSize()}");
8. }
9.
10.// Iteration through list using foreach
11.foreach (Circle circle in circlesList)
12.{
13.WriteLine($"Area Size = {circle.GetAreaSize()}");
14.}
15.
16.// List clearance
17.circlesList.Clear();
